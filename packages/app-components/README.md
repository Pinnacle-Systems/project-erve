# @erve/app-components

## Purpose

Provides business-neutral application components composed from primitives.

## What Belongs Here

- Page headers, filter bars, status summaries, empty states, loading states, error states, confirm dialogs, and generic panels.

## What Does Not Belong Here

- Client dashboards.
- Module-specific screens.
- Posting, pricing, tax, allocation, or approval hierarchy logic.

## Allowed Dependencies

- `@erve/primitives`
- `@erve/tokens`

## Examples of Future Exports

- `PageHeader`
- `FilterBar`
- `StatusBadge`
- `EmptyState`
